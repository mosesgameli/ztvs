# Security Policy (ZTVS)

Zero Trust Vulnerability Scanner (ZTVS) is committed to providing a secure and isolated host scanning environment. We take security vulnerabilities seriously and appreciate your help in making our project safer.

## Reporting a Vulnerability

If you discover a security vulnerability within ZTVS, please do not disclose it publicly. Instead, report it through one of the following channels:

*   **Email**: [ztvs-security@mosesgameli.com](mailto:ztvs-security@mosesgameli.com)
*   **GitHub**: Use the GitHub "Report a security vulnerability" feature (if enabled).

We will acknowledge receipt of your report within 48 hours and provide a timeline for resolution.

## Information to Include

To help us address the issue, please include as much information as possible:

*   Description of the vulnerability and its potential impact.
*   Steps to reproduce the issue.
*   Relevant versions of ZTVS, operating system, and hardware.
*   Any proof-of-concept (PoC) code or scripts.

## Supported Versions

Currently, we provide security updates for the following versions:

| Version | Supported |
| :--- | :--- |
| `main` | ✅ Yes |
| `< v1.0.0` | ❌ No |

## Responsible Disclosure

We ask that you follow these guidelines for responsible disclosure:

1.  Give us reasonable time to investigate and resolve the issue before making any information public.
2.  Do not attempt to access or modify data that does not belong to you.
3.  Do not use the vulnerability for any purpose other than reporting it to us.
4.  Compliance with these guidelines will be viewed as acting in good faith.

## Our Commitment

We commit to:

*   Investigating all legitimate reports promptly.
*   Fixing critical vulnerabilities as quickly as possible.
*   Acknowledging the researcher's contribution (with their permission) once the vulnerability is resolved.
